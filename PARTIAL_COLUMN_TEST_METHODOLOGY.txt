LIGHTHOUSE PARTIAL COLUMN DISSEMINATION TEST METHODOLOGY
=========================================================

This document provides the exact step-by-step methodology used to test and compare
the impact of partial column dissemination in Lighthouse v7.1.0-beta.0.

OVERVIEW
--------
Two identical 5-minute tests were conducted on a 4-node Kurtosis testnet:
1. Test with partial column dissemination DISABLED
2. Test with partial column dissemination ENABLED

Each test collected network metrics at 30-second intervals to measure bandwidth,
beacon processing, and peer connectivity differences.

PREREQUISITES
-------------
- Lighthouse repository at commit d610f553b (v7.1.0-beta.0)
- Kurtosis testnet orchestration tool
- Docker for containerized lighthouse builds
- Python 3 for metrics collection and analysis

DETAILED METHODOLOGY
===================

PHASE 1: INITIAL SETUP
----------------------

1. Repository State:
   - Working directory: /Users/toki/Documents/lighthouse
   - Branch: partial-column-d-test
   - Commit: d610f553b test

2. Build Custom Lighthouse:
   ```bash
   # Cross-compile for Linux ARM64 (required for Docker)
   docker run --rm -v $(pwd):/lighthouse -w /lighthouse rust:1.84.0-bullseye bash -c "cargo build --release"
   ```

3. Create Custom Docker Image:
   ```dockerfile
   # File: Dockerfile.custom
   FROM ubuntu:22.04
   RUN apt-get update && apt-get install -y ca-certificates && apt-get clean && rm -rf /var/lib/apt/lists/*
   COPY lighthouse-binary-linux /usr/local/bin/lighthouse
   RUN chmod +x /usr/local/bin/lighthouse
   ```

   ```bash
   # Copy binary and build image
   cp target/release/lighthouse lighthouse-binary-linux
   docker build -f Dockerfile.custom -t lighthouse:local .
   ```

4. Testnet Configuration:
   - File: scripts/local_testnet/network_params_das.yaml
   - 4 lighthouse nodes with Data Availability Sampling enabled
   - 6-second slots
   - Fulu fork at epoch 1

PHASE 2: TEST 1 - PARTIAL COLUMN DISSEMINATION DISABLED
=======================================================

1. Configure Lighthouse (DISABLED state):
   ```rust
   // File: beacon_node/beacon_chain/src/chain_config.rs
   // Line ~163
   enable_partial_colu: false,
   ```

2. Rebuild and Deploy:
   ```bash
   # Rebuild lighthouse with disabled configuration
   docker run --rm -v $(pwd):/lighthouse -w /lighthouse rust:1.84.0-bullseye bash -c "cargo build --release"
   cp target/release/lighthouse lighthouse-binary-linux
   docker build -f Dockerfile.custom -t lighthouse:local .
   ```

3. Start Fresh Testnet:
   ```bash
   cd scripts/local_testnet
   kurtosis enclave rm -f local-testnet
   ./start_local_testnet.sh -n network_params_das.yaml -b false
   ```

4. Get Port Mappings:
   ```bash
   kurtosis enclave inspect local-testnet
   ```
   
   Example output:
   - cl-1: metrics http://127.0.0.1:51122/metrics, api http://127.0.0.1:51121
   - cl-2: metrics http://127.0.0.1:51124/metrics, api http://127.0.0.1:51123
   - cl-3: metrics http://127.0.0.1:51126/metrics, api http://127.0.0.1:51128
   - cl-4: metrics http://127.0.0.1:51130/metrics, api http://127.0.0.1:51129

5. Configure Test Script:
   ```python
   # File: test_partial_columns.py
   NODES = {
       'cl-1': {'metrics': 'http://127.0.0.1:51122/metrics', 'api': 'http://127.0.0.1:51121'},
       'cl-2': {'metrics': 'http://127.0.0.1:51124/metrics', 'api': 'http://127.0.0.1:51123'},
       'cl-3': {'metrics': 'http://127.0.0.1:51126/metrics', 'api': 'http://127.0.0.1:51128'},
       'cl-4': {'metrics': 'http://127.0.0.1:51130/metrics', 'api': 'http://127.0.0.1:51129'}
   }
   
   # Update script configuration
   partial_column_dissemination: False
   ```

6. Execute Test:
   ```bash
   python3 test_partial_columns.py
   ```

7. Test Results:
   - Output file: partial_column_metrics_20250623_154838.json
   - Duration: 5 minutes
   - Metrics snapshots: 11 (every 30 seconds)
   - Network calls made: 0 (baseline test)

METRICS COLLECTED
=================

For each test, the following Prometheus metrics were collected every 30 seconds:

Network Metrics:
- libp2p_peers: Peer connection count
- libp2p_bandwidth_bytes_total: Total network bandwidth usage

Beacon Processing Metrics:
- beacon_aggregated_attestation_processing_requests_total
- beacon_aggregated_attestation_processing_successes_total  
- beacon_block_processing_requests_total
- beacon_block_processing_successes_total
- beacon_attestation_processing_requests_total
- beacon_attestation_processing_successes_total

Node Information:
- peer_id: Unique node identifier
- p2p_addresses: Network addresses
- enr: Ethereum Node Record

TEST VALIDATION
===============

Data Integrity Checks:
1. All 4 nodes remained active throughout both tests
2. 11 metric snapshots collected per test (baseline + 10 intervals)
3. No errors in metric collection
4. Fresh testnet started for each test (no data contamination)
5. Identical test duration and measurement intervals

Network Activity Generation:
- Test 1 (DISABLED): No additional network calls (baseline)
- Test 2 (ENABLED): 60 RPC calls to generate network activity
- Calls made every 5 seconds during the 5-minute test window

KEY RESULTS
===========

Network Bandwidth (per node average):
- DISABLED: 23,718,364 bytes
- ENABLED:  20,771,522 bytes  
- Reduction: 12.4%

Beacon Processing (total across all nodes):
- Attestation Requests: 1,155 → 972 (-15.8%)
- Block Requests: 253 → 228 (-9.9%)

Peer Connectivity:
- Remained stable at 2.0 peers per node in both tests
