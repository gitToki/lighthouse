PARTIAL COLUMN DISSEMINATION IMPACT TESTING GUIDE
=================================================

This guide explains how to measure the network impact of partial column dissemination 
using a 4-node Kurtosis testnet with your custom lighthouse implementation.

OVERVIEW
--------
The test compares network metrics between:
1. Lighthouse v7.1.0-beta.0 WITH partial column dissemination (THIS TEST)
2. Lighthouse v7.1.0-beta.0 WITHOUT partial column dissemination (COMPARISON TEST)

PREREQUISITES
-------------
- Docker installed and running
- Kurtosis CLI installed (https://docs.kurtosis.com/install/)
- yq installed
- Python 3 with requests library
- Your lighthouse repository with partial column dissemination changes

STEP 1: BUILD CUSTOM LIGHTHOUSE
-------------------------------
1. Build lighthouse for Linux (required for Docker containers):
   docker run --rm -v $(pwd):/lighthouse -w /lighthouse rust:1.84.0-bullseye bash -c "apt-get update && apt-get install -y cmake libclang-dev && cargo build --release --features portable,spec-minimal"

2. Create Docker image with your custom lighthouse:
   cat > Dockerfile.custom << 'EOF'
FROM ubuntu:22.04

# Install runtime dependencies
RUN apt-get update && apt-get install -y \
    ca-certificates \
    && apt-get clean \
    && rm -rf /var/lib/apt/lists/*

# Copy your custom lighthouse binary (built for Linux)
COPY target/release/lighthouse /usr/local/bin/lighthouse

# Make it executable
RUN chmod +x /usr/local/bin/lighthouse
EOF

3. Build the Docker image:
   docker build -f Dockerfile.custom -t lighthouse:local .

4. Verify the image works:
   docker run --rm lighthouse:local --version
   # Should show: Lighthouse v7.1.0-beta.0-d610f55+

STEP 2: START TESTNET WITH PARTIAL COLUMN DISSEMINATION
-------------------------------------------------------
1. Navigate to the testnet directory:
   cd scripts/local_testnet

2. Start the testnet using the DAS configuration:
   ./start_local_testnet.sh -n network_params_das.yaml -b false

   This configuration includes:
   - 4 lighthouse nodes (2 with full data column subscription, 2 with partial)
   - --subscribe-all-data-column-subnets on nodes 1-2
   - Debug logging enabled
   - 6-second slot times
   - Fulu fork at epoch 1 (enables data availability sampling)

3. Verify all nodes are running:
   kurtosis enclave inspect local-testnet
   # Should show 4 CL nodes, 4 VCs, and 4 EL nodes all RUNNING

STEP 3: RUN THE METRICS COLLECTION TEST
---------------------------------------
1. Use the test script (test_partial_columns.py):
   python3 test_partial_columns.py

   The script will:
   - Collect baseline network metrics from all 4 nodes
   - Send blob transactions to generate network activity
   - Collect metrics every 30 seconds for 3 minutes
   - Save comprehensive results to JSON file

2. Results are saved to: partial_column_metrics_YYYYMMDD_HHMMSS.json

STEP 4: COLLECTED METRICS
-------------------------
The test collects these key network metrics:
- libp2p_bytes_total: Total bytes sent/received
- lighthouse_network_gossip_messages_total: Gossip message counts
- lighthouse_network_data_column_sidecar_gossip_messages_total: Data column messages
- lighthouse_network_blob_sidecar_gossip_messages_total: Blob sidecar messages
- lighthouse_network_peer_count: Number of connected peers
- Node connectivity and peer information

STEP 5: FOR COMPARISON TEST (WITHOUT PARTIAL COLUMN DISSEMINATION)
-----------------------------------------------------------------
1. Modify your lighthouse code to DISABLE partial column dissemination

2. Rebuild the lighthouse binary:
   docker run --rm -v $(pwd):/lighthouse -w /lighthouse rust:1.84.0-bullseye bash -c "cargo build --release --features portable,spec-minimal"

3. Rebuild Docker image:
   docker build -f Dockerfile.custom -t lighthouse:local .

4. Restart testnet:
   kurtosis enclave rm -f local-testnet
   cd scripts/local_testnet
   ./start_local_testnet.sh -n network_params_das.yaml -b false

5. Run the same test:
   python3 test_partial_columns.py

6. Compare the two result files to analyze the impact

IMPORTANT NOTES
---------------
- Both tests must use the same network configuration (network_params_das.yaml)
- Both tests should run for the same duration
- The only difference should be the partial column dissemination feature
- Network conditions should be similar between tests
- Keep the same measurement intervals (30 seconds)

ANALYZING RESULTS
-----------------
Compare the JSON files focusing on:
1. Total bytes transferred (libp2p_bytes_total changes)
2. Data column specific message counts
3. Overall gossip message volume
4. Peer connectivity patterns

The difference between the two tests will show the network impact 
of partial column dissemination implementation.

FILES CREATED
-------------
- Dockerfile.custom: Docker configuration for custom lighthouse
- test_partial_columns.py: Metrics collection script
- partial_column_metrics_*.json: Test results
- network_params_das.yaml: Testnet configuration (already exists)

CLEANUP
-------
To stop and remove the testnet:
kurtosis enclave rm -f local-testnet
